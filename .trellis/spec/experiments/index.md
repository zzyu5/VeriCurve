# Experiments Spec

## Pre-Development Checklist

- Identify whether the task is Experiment 0, 1, 2, 3, controller evaluation, or paper analysis.
- Read [go-nogo](./go-nogo.md) before claiming direction strength.
- Use [artifacts](./artifacts.md) for result naming and evidence requirements.

## Guidelines

| Guide | Description |
|---|---|
| [go-nogo](./go-nogo.md) | Mandatory Go/No-Go sequence and criteria. |
| [artifacts](./artifacts.md) | Result files, command logs, and metadata requirements. |
| [workloads](./workloads.md) | Workload categories and acceptance-drift expectations. |

## Quality Check

- Every result must be reproducible from recorded commands.
- Any speedup claim must include the baseline.
- Any failure should be preserved as evidence, not erased.
